============
Mulitprocessing
============

To run multiprocessing with :code:`redback`, the built-in Python:code:`multiprocessing` package. Multiprocessing can be enabled in the :code:`redback.fit_model` function by setting the :code:`n_pool` parameter, which specifies the number of parallel worker processes to use.


Linux
-------------------------

On Linux-based systems, multiprocessing can be enabled simply by specifying the desired number of processes:

.. code:: python

import multiprocessing

..........

result = redback.fit_model(model=model, 
							sampler='nessai', 
							transient=transient, 
							nlive = 1000,
                            prior=priors,
                            model_kwargs=model_kwargs,
                            n_pool=4)


In this example, :code:`n_pool=4` creates a pool of four worker processes that evaluate the likelihood in parallel.


macOS
-------------------------

Recent versions of macOS default to the "spawn" start method for multiprocessing. While "spawn" is safer for certain system-level interactions, it starts a completely fresh Python interpreter for each worker process. This can cause issues when working with multiprocessing in :code:`redback`. Instead, we add a line that explicitly sets the multiprocessing start method to "fork". A :code:`if __name__ == "__main__":` frame is also required in this instance to prevent infinite process spawning, ensuring that the code is only run in the original process. 

For MacOS machines, the line changing the start method to "fork" should be placed at the start of the script, while the :code:`if __name__ == "__main__":` frame is placed around the :code:`redback.fit_model` function.


.. code:: python

import multiprocessing
multiprocessing.set_start_method("fork", force=True)

if __name__ == "__main__":
